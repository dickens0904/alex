# 项目开发全局规则（Trae AI必须严格遵守）

## 一、基础编码规范
- 编程语言：Python 3.11+
- 编码规范：严格遵循PEP 8
- 变量命名：snake_case（变量/函数/模块），PascalCase（类），UPPER_SNAKE_CASE（常量）
- 行长度：最大120字符
- 导入顺序：标准库→第三方库→本地应用，每组之间空一行
- 注释要求：
  - 所有公共函数/类/方法必须有Google风格的文档字符串
  - 文档字符串必须包含：功能描述、参数、返回值、异常说明
  - 复杂逻辑必须添加行内注释说明意图
- 错误处理：
  - 禁止使用裸露的`except:`，必须捕获具体异常类型
  - 异常信息必须包含足够的上下文便于调试
  - 资源操作必须使用with语句确保正确释放

## 二、技术栈与依赖约束
- Web框架：FastAPI 0.100+
- 数据库：PostgreSQL 15 + SQLAlchemy 2.0（异步模式）
- ORM：禁止使用原始SQL，必须通过SQLAlchemy操作数据库
- 依赖管理：Poetry，版本号使用^兼容模式
- 测试框架：pytest + pytest-asyncio
- 代码质量：black（格式化）+ isort（导入排序）+ mypy（类型检查）+ flake8（静态分析）
- 日志：使用项目统一的logger，**绝对禁止**使用print语句
- 配置：所有配置从环境变量或.env文件读取，禁止硬编码

## 三、代码结构与组织
- 项目采用分层架构：api→service→repository→model
- 单个函数长度不超过50行，单个类不超过300行
- 单个文件不超过500行
- 每个模块职责单一，避免出现"万能模块"
- 循环依赖必须通过依赖注入解决

## 四、数据库规范
- 所有表必须有id（UUID主键）、created_at、updated_at字段
- 外键必须添加索引
- 禁止使用SELECT *
- 批量操作必须使用bulk_insert/bulk_update
- 数据库迁移必须使用Alembic，禁止手动修改数据库结构

## 五、API设计规范
- RESTful风格，资源使用复数形式
- HTTP方法语义正确：GET（查询）、POST（创建）、PUT（全量更新）、PATCH（部分更新）、DELETE（删除）
- 响应格式统一：
  ```json
  {
    "code": 0,
    "message": "success",
    "data": {}
  }
  ```
- 错误码统一管理，使用HTTP状态码+业务错误码
- 所有接口必须有OpenAPI文档说明

## 六、安全要求
- 所有用户输入必须使用Pydantic进行验证和清洗
- 密码必须使用bcrypt哈希存储，迭代次数不低于12
- API接口必须进行JWT认证和权限验证
- 禁止在日志中输出敏感信息（密码、token、手机号、身份证号）
- 防止SQL注入、XSS、CSRF等常见攻击
- 所有外部接口调用必须设置超时时间

## 七、测试要求
- 单元测试覆盖率不低于80%
- 核心业务逻辑覆盖率必须达到100%
- 测试用例必须覆盖正常情况、边界情况和异常情况
- 每个函数至少有一个测试用例
- 测试代码与生产代码结构保持一致

## 八、Trae AI行为准则（最高优先级）
1. 当你不确定时，**永远提问而不是猜测**
2. 生成代码前必须先理解现有代码的结构和风格
3. 不要修改你不理解的代码
4. 所有修改必须有明确的理由
5. 如果发现代码中有bug，优先修复而不是绕过
6. 严格遵守本文件中的所有规则，如有冲突以本文件为准
7. 不要生成任何与当前任务无关的代码
8. 生成的代码必须能够直接运行，不能有语法错误
